🔤 Abbreviations

URSP: UE Route Selection Policy
TD: Traffic Descriptor
RSD: Route Selection Descriptor
PTI: Procedure Transaction Identity
PLMN: Public Land Mobile Network
UPSC: UE Policy Section Contents
PV: Precedence Value (Lower = Higher Priority)

🏗️ URSP Rule Structure

URSP Rule Components

  • Basic Info: Precedence Value (Priority)
  • Traffic Descriptor: Traffic identification components
  • Route Selection Descriptor: Network routing components

Match-all Rule

  • Automatically added with highest PV (lowest priority)
  • Catches all traffic not matched by other rules
  • Essential for complete policy coverage

🎯 TD Validation Rules

Same Type Components

  • Logic: OR (At least one must match)
  • Example 1: DNN="internet" OR DNN="ims"
  • Example 2: Connection Capabilities (IMS OR MMS OR SUPL)

Different Type Components

  • Logic: AND (All must match)
  • Example: DNN="internet" AND App="com.example"

Mutually Exclusive Types

  • Logic: NOT Allowed (Standard restriction)
  • Reason: 3GPP TS 24.526 Table 5.2.1 NOTE 6: the receiving entity shall ignore the URSP rule
  • Example: Single remote port + Remote port range

🛣️ RSD Validation Rules

Same Type Components

  • Logic: NOT Allowed (Tool recommendation)
  • Reason: Causes UE implementation-dependent routing selection. Single value recommended for predictable routing.
  • Example: Cannot have multiple DNN values in one RSD

Different Type Components

  • Logic: AND (All must match)
  • Example: S-NSSAI + DNN + SSC Mode

Routing Fallback Options

  • Method: Multiple RSDs with different precedence values
  • Priority: Lower precedence value = Higher priority
  • Example: RSD(PV=1) tried first, RSD(PV=2) as fallback
